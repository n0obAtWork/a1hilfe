# Kachel einrichten

<!-- source: https://amic.de/hilfe/kacheleinrichten.htm -->

Administration > Menü > Dashboard > Variante Kachel

oder

Direktsprung **[DASH]** \> Variante Kachel

Auf einem bereits eingerichteten Dashboard erreicht man die Bearbeitungsmaske der Kachel auch direkt über das Kontextmenü (rechte Maustaste) des Dashboards, wenn man mit der Maus über der Kachel steht.

![Ein Bild, das Text enthält. Automatisch generierte Beschreibung](../../../ImagesExt/image8_1489.png "Ein Bild, das Text enthält. Automatisch generierte Beschreibung")

| Feldbezeichnung | Beschreibung |
| --- | --- |
| Titel | Der Titel dient als Beschreibung der Kachel. Dieser muss eindeutig sein. Zusätzlich wird eine Kachel intern über eine eindeutige Ident identifiziert. Der Titel kann jederzeit geändert werden.<br> |
| Darstellungsart | Art der Darstellung. Es existieren folgende Möglichkeiten:<br><br><br>• [Text](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_text.md)<br>• [Tabelle](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_tabelle.md)<br>• [Fortschrittsbalken](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_fortschrittsbalken.md)<br>• [Skala](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_skala.md)<br>• [Säulendiagramm](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_saeulen_flaechen_und_liniendiagramm.md)<br>• [Flächendiagramm](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_saeulen_flaechen_und_liniendiagramm.md)<br>• [Liniendiagramm](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_saeulen_flaechen_und_liniendiagramm.md)<br>• [Tortendiagramm](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_tortendiagramm.md)<br>• [Bild](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_bild.md)<br>• [Deutschlandkarte](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_deutschland_europakarte.md)<br>• [Europakarte](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_deutschland_europakarte.md)<br>• [Kombinationsdiagramm](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_kombinationsdiagramm.md)<br>• [Balkendiagramm](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_balkendiagramm.md)<br>• [Kalender](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_kalender.md)<br> |
| View/Prozedur | Über die private Prozedur oder View werden alle Daten geliefert, die zur Erstellung einer Kachel benötigt werden. Dabei unterscheiden sich die benötigten Daten je nach Darstellungsart. Eine genaue Beschreibung der verschiedenen Darstellungsarten findet man unter „[Prozeduren oder Views für Kacheln einrichten](./prozeduren_oder_views_fuer_kacheln_einrichten/index.md)“.<br>Grundsätzlich werden Views bzw. Prozeduren ohne Parameter aufgerufen. Man der Prozedur jedoch eigene Parameter mitgeben. Dies können Konstanten, db_variablen oder LDB-Variablen (mit vorangestelltem Doppelpunkt) sein. Beispiel<br> <br><pre><code>Create procedure p_dash_kachel(in integer&#10; bedienerid)&#10;.</code></pre><br> <br>![](../../../ImagesExt/image8_1490.png)<br> |
| Refresh aktiv | Hiermit wird gekennzeichnet, ob diese Kachel auf eine „Refresh-Prozedur“ reagieren soll (Refresh aktiv = Ja) oder nicht.<br> |
| <br>**Bei Klick ausführen:** Die folgenden Felder sind **nur alternativ** zu belegen. Sind mehrere Felder belegt, so wird nur die erste belegte Funktion ausgeführt. Steht der Mauszeiger auf einer Kachel mit einer Funktion, so wird das Handsymbol als Mauszeiger verwendet. |
| Funktion | Hier kann eine private oder offizielle Anwendungsfunktion (Direktsprung [ANWF]) eingetragen werden, die als Direktsprung oder aus dem Menü heraus aufgerufen werden kann. In den Funktionen kann auf die Felder **ID1** bis **ID4** über die JVars **JVAR_KACHEL_ID1** bis **JVAR_KACHEL_ID4** mit dem Owner 7659 zugegriffen werden:<br> |
| Pfleger | Aufruf eines Stammdatenpflegers, der im Pflegerstamm definiert sein muss. Für diesen Funktionsaufruf müssen die Werte, die für ID1, ID2, ID3 bzw. ID4 erwartet werden, von der View geliefert werden.<br>Wenn man hier z.B. den Kundenstamm als Pfleger aufrufen möchte, so müsste in der View u.a. stehen:<br> <br><pre><code>Select kundid as ID1,&#10;…..</code></pre><br>Ein Beispiel findet man auch unter [Geographische Karte](./prozeduren_oder_views_fuer_kacheln_einrichten/darstellungsart_deutschland_europakarte.md)<br> |
| Refresh-Prozedur | Hier kann eine Prozedur angegeben werden, mit der einzelne Kacheln des Dashboards neu angezeigt werden können. Diese Prozedur soll die idents der Kacheln zurückgeben, die neu Aufgebaut werden sollen. Dabei ist die Idee dahinter die, dass durch den Klick ein Wert in eine Datenbankvariable gesetzt wird, der dann von den anderen Kacheln ausgewertet wird. Damit kann man dann zu dem ausgewählten Datensatz noch genauere Informationen anzeigen. <br>Die Datenbankprozedur hat genau sechs Parameter, die Id_board des Dashboards, die Id_kachel der auslösenden Kachel und die Identwerte, die ID1 bis ID4 entsprechen, wie sie auch vom Pfleger (siehe oben) verwendet werden. Der Typ der Ident-Parameter (interger, string, …) hängt vom Typ des ID-Feldes ab.<br>Sie muss die Kachel_id der Kacheln liefern, die neu aufgebaut werden sollen.<br> <br>Beispielprozedur:<br><pre><code>CREATE PROCEDURE&#10; p_dash_Refresh&#10; (in in_id_board integer,&#10; in&#10; in_id_kachel integer,&#10; in&#10; in_ident1 integer default null,&#10; in&#10; in_ident2 integer default null,&#10; in&#10; in_ident3 integer default null,&#10; in&#10; in_ident4 integer default null )&#10; &#10;BEGIN&#10; create or&#10; replace Variable pdb_adressid integer =0;&#10; set pdb_adressid =&#10; in_ident1;&#10;&#10; select id_kachel&#10; from Dash_Board_Kachel_Link&#10; where Id_Board&#10; = in_id_board and id_kachel!= in_id_kachel;&#10; EXCEPTION&#10; when others&#10; then&#10; &#10; call amic_exception( ERRORMSG() &#124;&#124;&#10; CHAR(10) &#124;&#124; CHAR(13) &#124;&#124; TRACEBACK(), SQLCODE , SQLSTATE , 'p_dash_Refresh'&#10; , -1 );&#10;END</code></pre><br>In den privaten Prozeduren der Kacheln würde man dann auf pdb_adressid zugreifen:<br><br>Beispielprozedur einer einfachen Textkachel (ohne jegliche Formatierung):<br><pre><code>CREATE&#10; PROCEDURE p_dash_kundeninfo()&#10;BEGIN&#10; &#10; if&#10; varexists('pdb_adresssid') = 0&#10; then&#10; create or&#10; replace variable pdb_adresssid&#10; integer = -1;&#10; endif;&#10; select&#10;&#10; &#10; adressname &#124;&#124; '&lt;br&gt;' &#124;&#124; adressStrasse &#124;&#124; '&lt;br&gt;' &#124;&#124; adressplz1&#10; &#124;&#124;' ' &#124;&#124; adressort as text&#10; &#10;&#10; from anschriftstamm where adressid =&#10; pdb_adresssid and adresstyp in&#10; (11,12,15)&#10;END</code></pre><br> |
